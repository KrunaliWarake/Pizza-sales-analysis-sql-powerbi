# Pizza Sales Analysis (MySql, PowerBi)

---

## Overview

This project demonstrates an end-to-end data analytics workflow using <b>*MySQL*</b> and <b>*Power BI*</b> to analyze pizza sales data and generate actionable business insights.

The goal is to transform raw transactional data into meaningful insights related to *sales performance, customer behavior, and product trends*.

---

## Objectives

* Analyze key performance indicators (KPIs) such as revenue, orders, and quantity
* Identify best and worst-performing pizzas
* Understand customer purchasing patterns
* Evaluate sales trends across time, categories, and sizes
* Build an interactive dashboard for analysis

---

## Tech Stack

* *MySQL* – Data cleaning, transformation, and querying
* *Power BI* – Data visualization and dashboard development
* *DAX* – Measures and calculations
* *CSV Dataset* – Source data

---

##  Workflow

1. Imported raw dataset into MySQL
2. Performed data cleaning and preprocessing
3. Wrote SQL queries to calculate KPIs and insights
4. Connected data to Power BI
5. Built an interactive dashboard with filters and visuals

---

## Key Metrics

* *Total Revenue:* 817.86K
* *Total Orders:* 21K
* *Total Pizzas Sold:* 50K
* *Average Order Value:* 38.31
* *Average Pizzas per Order:* 2.32

---

##  Key Insights

###  Sales Trends

* Highest sales on *Fridays and Saturdays*
* Peak order time during *evening hours*
* *July and January* have maximum sales

###  Product Performance

* *Top Revenue Pizza:* Thai Chicken Pizza
* *Most Ordered Pizza:* Classic Deluxe Pizza
* *Lowest Performing Pizza:* Brie Carre Pizza

### Category & Size Analysis

* *Best Category:* Classic
* *Most Popular Size:* Large

---

## Dashboard Features

* KPI cards for quick insights
* Daily and monthly sales trends
* Top and bottom selling pizzas
* Category-wise and size-wise analysis
* Interactive filters for dynamic exploration
